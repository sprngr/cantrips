# Restore Playbook

Load this file when `squash-unpack.sh` returns `"status":"error"` or fails schema/hash/char checks.

## Error handling

1. Identify error code from first JSON line.
2. Stop restore on any non-zero exit.
3. Discard payload when integrity fails.
4. Re-pack from original source before retry.

## Code: `sha256_mismatch`

1. Delete tampered payload file.
2. Re-run `bash scripts/squash-pack.sh --input "<context-file>"`.
3. Re-run `bash skills/experimental/squash/scripts/squash-unpack.sh --file "<new-payload>"`.

## Code: `schema_missing_keys`

1. Verify envelope uses required keys from `assets/context-envelope.schema.json`.
2. Re-generate payload with current `scripts/squash-pack.sh`.
3. Re-run unpack only after schema fields match.

## Code: `char_count_mismatch`

1. Treat payload as corrupted.
2. Re-pack from untouched source file.
3. Re-run unpack and verify `context_chars` again.

## Code: `gzip_read_failed` or `json_decode_failed`

1. Confirm payload file path points to `.json.gz` generated by squash.
2. Re-pack source file.
3. Re-run unpack with newly generated payload.
